Cursor 里的对话历史由客户端管理，**本仓库无法自动写入完整聊天记录**。若需本地留档，请你在 Cursor 中：

- 手动复制对话到本地 `.txt` / `.md`；  
- 或将本文件与下文「对话摘要」一并保存。

## 三、对话摘要（2026-04 前后，技术结论 · 大白话）

1. **全站只剩「无法获取 Notion 数据」**  
   - 根因之一已证实：`lib/notion/getAllPageIds.js` 的 `catch` 里误用了变量 **`ids`**，触发 `ReferenceError`，拖垮 `getGlobalData`。已修复。  
   - 排障应优先看 **Vercel 日志带堆栈**，再改代码。

2. **以后修 Bug 的流程（用户要求）**  
   - 修订代码前**先索要证据**（日志 / `[NOTION_SITE_DATA]` 等），避免无依据乱改。已写入 `project-context.mdc`。

3. **缓存**  
   - `site_data_*` 占位不应长期当有效缓存；`getDataFromCache` 与生产环境行为已对齐说明，见主规则文件。

4. **可观测性**  
   - 日志前缀 `[NOTION_SITE_DATA]`；可选 `SITE_DIAG_SECRET` + `GET /api/health-site`（见主规则）。

5. **首页列表图加速**  
   - `HOME_LIST_COVER_MAX_WIDTH`：**680 → 480**（仅列表/分页卡片；详情大图仍走 `IMAGE_COMPRESS_WIDTH`）。

6. **REDIS**  
   - 未配置时与多数占位问题无关；按需再加。
